I. Architectural Vision

Nexus Radio is a high-performance communication node designed to bridge web-based P2P protocols with native Android hardware. It eliminates the distinction between display hardware and software, functioning as a persistent "firmware overlay."

NexusBridge v2.0

Native bi-directional Java interface allowing the Web UI to control Bluetooth, Battery, and System Settings directly from window.NexusBridge.

AI Voice Trigger

Integrated Voice Activity Detection (VAD) with 1500ms hysteresis. Automatically manages bandwidth by toggling transmission based on speech.

II. Protocol & Security

Bypass Logic

The system is specifically engineered to run in restricted environments. It automatically detects the presence of NexusBridge or the file:// protocol to:

  • Bypass SSL (HTTPS) Requirements: Disables browser warnings for insecure contexts, enabling full camera/mic access on local files.
  • Dynamic URL Generation: Uses the getBaseURL() helper to provide valid https://vdo.ninja/ fallbacks when location.host is empty.
  • Secure Salt Initialization: Automatically sets session.salt to vdo.ninja when running from storage.

III. Hardware Diagnostics

Real-time telemetry is available via NexusBridge.getDiagnostics():

  • Sensors: High-frequency access to Accelerometer, Gyroscope, and Magnetometer data.
  • Power: Monitoring of battery level, charging state, and remaining capacity.
  • Display: Pixel-perfect rendering utilizing DisplayMetrics scale factors.

IV. Build & Deployment

Nexus Radio follows an autonomous build pipeline:

  1. Asset Sync: Web files from /web-ui-server are compiled into /android-launcher/app/src/main/assets/.
  2. Compilation: Build using Gradle 8.5+ with Java 11/21 requirement.
    ./gradlew assembleDebug
  3. Deployment: Push directly to device via ADB.
    adb install -r nexus-radio.apk

V. Tech Stack

  • Frontend: Modified VDO.Ninja Core (Vanilla JS / Web Audio API)
  • Embedded: Android Pie (API 28+) / Java Native Bridge
  • Protocol: WebRTC P2P / WHEP / WHIP
  • AI: Local threshold-based VAD
